# Content Rules

## 内容取舍

只写 demo 中会被玩家看到或被 Godot 读取的内容。背景设定必须服务当前闭环，不能为了完整世界观而扩写系统。

## 文案规则

- 对话像真人说话，少用书面套话。
- NPC 关注自己的生计、规矩和麻烦。
- 场景描写写具体物件，不写抽象意境。
- 任务描述短、直接、有明确目标。
- 不使用宿命、绝世、天命、救世等高强度词。

## 禁止内容

- 不新增门派任务。
- 不新增战斗任务。
- 不新增隐藏武功或秘籍。
- 不新增超出清风镇的可玩地点。
- 不新增超过 demo 数据契约的复杂条件表达式。

## 命名规则

- 地点名朴素具体：林家药铺、清风客栈、溪边草坡。
- NPC 名称短而生活化：药师林、周掌柜、赵守卫。
- id 使用 snake_case，带类型前缀：`npc_herbalist_lin`、`quest_collect_herbs`。
