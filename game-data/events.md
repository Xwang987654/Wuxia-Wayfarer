# 事件 / 剧情

## 概述

事件是推动游戏世界运转的引擎。事件分为定时事件、随机事件和条件触发事件。

## 事件分类

| 类型 | 触发方式 | 示例 |
|------|---------|------|
| 定时事件 | 固定时间触发 | 季节变化、节日 |
| 随机事件 | 随概率触发 | 路遇旅人、发现宝箱 |
| 条件事件 | 满足条件触发 | 好感达标、声望达标 |
| 探索事件 | 进入特定区域触发 | 古庙探秘、后山遇贼 |

## 事件系统设计

### 事件结构

```yaml
事件ID: event_random_001
名称: 路遇旅人
类型: 随机
触发区域: 竹林小径
触发概率: 20%
描述: 一个疲惫的旅人坐在路边，看起来需要帮助。

选择:
  - 文本: "上前帮助"
    条件: 无
    结果:
      好感: 旅人好感+20
      声望: 侠义+5
      奖励: 获得"旅人的谢礼"（随机物品）

  - 文本: "绕道而行"
    条件: 无
    结果: 无

  - 文本: "趁火打劫"
    条件: 无
    结果:
      声望: 恶名+10
      奖励: 获得50铜钱
      后续: 旅人记住你，未来可能报复
```

### 事件队列

- 同一时间最多触发1个事件
- 事件有冷却时间（同一事件24小时内不重复触发）
- 优先级：条件事件 > 定时事件 > 随机事件
