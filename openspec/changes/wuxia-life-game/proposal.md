# 武侠·自由人生 — 游戏提案

> 一款养生向武侠模拟经营游戏，强调自由度、NPC互动与人生路线多样性。

---

## 一、游戏概述

### 定位
- **类型**：武侠模拟经营 + 养成 + 轻度探索
- **风格**：养生不肝、低操作需求、随机性驱动
- **视角**：2.5D 斜俯视（45度等距视角）
- **画风**：像素风，AI辅助生成美术素材
- **参考**：烟雨江湖（但自由度更高、加入经营玩法）

### 核心差异化（对比烟雨江湖）

| 维度 | 烟雨江湖 | 本作 |
|------|---------|------|
| 剧情 | 固定主线剧情 | 自由选择人生路线 |
| 战斗 | 战斗为核心玩法 | 经营/社交与战斗并重 |
| NPC | 功能性为主 | 深度交互：交易、社交、关系 |
| 经济 | 简单货币系统 | 交易/开店/跑商 |
| 自由度 | 线性任务为主 | 多路线可自由切换 |

## 二、核心玩法

### 2.1 每日循环

玩家每天（10分钟现实时间）可以选择做以下事情：

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  经营赚钱    │────▶│  修炼武功    │────▶│  探索/事件   │
│  开铺子      │     │  读书悟道    │     │  随机遭遇    │
│  做买卖      │     │  拜师学艺    │     │  江湖传闻    │
└─────────────┘     └─────────────┘     └─────────────┘
       │                                       │
       │            ┌─────────────┐            │
       └───────────▶│   NPC社交    │◀───────────┘
                    │  交易/对话   │
                    │  关系/好感   │
                    └─────────────┘
```

### 2.2 人生路线（可自由选择、可切换）

玩家开局后可自由选择或在游戏过程中切换人生路线，路线之间不互斥：

#### 江湖散人
- 自由探索地图
- 接悬赏任务赚钱
- 行商跑买卖
- 声望系统（侠义/恶名）

#### 门派弟子
- 拜入武馆/门派
- 学习专属武功
- 完成门派任务、参加比武
- 成长路径：弟子 → 长老 → 掌门

#### 衙门捕快
- 入职衙门
- 查案、断案、维护治安
- 领取官府俸禄
- 官府专属任务线

#### 经商路线
- 开铺子（药铺/铁匠/茶馆等）
- 跑商（利用不同地点的价差）
- 炼丹/锻造出售
- 目标：富甲一方

**路线组合示例**：你可以是衙门捕快，同时在镇上开了间药铺，空闲时还去山里采药——完全自由。

## 三、战斗系统

### 设计原则
- 不需要操作、不硬核
- 核心乐趣在**战前策略**（装备搭配、武功选择、丹药准备）
- 战斗过程自动进行，以文字形式展示

### MVP 版本（初版 Demo）

第一版不做战斗场景，直接输出战斗结果，配以好看的 UI：

```
┌─────────────────────────────────────────────────┐
│  ⚔️  遭遇：山贼 x2                               │
│                                                  │
│  第1回合：你使出铁砂掌，击中山贼甲（-28HP）        │
│  第1回合：山贼乙挥刀砍来，你格挡（-12HP）          │
│  第2回合：你一脚踢中山贼乙要害（-45HP，暴击！）    │
│  第2回合：山贼甲逃跑                              │
│                                                  │
│  ──────────────────────────────────────          │
│  胜利！                                          │
│                                                  │
│  获得：铜钱 x80  粗布衣 x1  山贼腰牌 x1          │
│  消耗：生命 -12  体力 -5                         │
│                                                  │
│  [确认]                                          │
└─────────────────────────────────────────────────┘
```

### 战斗公式（草案）
- 伤害 = 攻击力 × 武功倍率 - 防御力 × 护甲系数 + 随机浮动
- 暴击：基于武功特性 + 随机概率
- 逃跑：基于轻功值 + 随机判定

### 后续扩展
- 完成 MVP 后可设计正式战斗页面（回合制 + 自动混合）
- 可中途干预（选择换招式/使用物品/逃跑）

## 四、时间系统

### 4.1 基础时间

| 参数 | 值 |
|------|-----|
| 1天（游戏内） | 10分钟（现实） |
| 1季 | 6天 = 1小时（现实） |
| 1年（四季循环） | 24天 = 4小时（现实） |

### 4.2 日夜时段

一天分为5个时段，每段2分钟（现实）：

| 时段 | 时间范围 | 影响 |
|------|---------|------|
| 清晨 | 0:00 - 2:00 | 修炼加成（吐纳最佳时机） |
| 上午 | 2:00 - 4:00 | 商铺开门，可交易 |
| 午后 | 4:00 - 6:00 | NPC活跃，可接任务 |
| 傍晚 | 6:00 - 8:00 | 集市热闹，稀有物品出现 |
| 夜晚 | 8:00 - 10:00 | 衙门关门，黑市开张，夜间事件 |

### 4.3 四季与天气

**四季特性：**

| 季节 | 特性 | 特殊事件 |
|------|------|---------|
| 春 | 草药生长旺盛 | 门派招弟子、春雨修炼加成 |
| 夏 | 烈日体力消耗快 | 江湖比武大会、商路繁荣 |
| 秋 | 丰收季产量翻倍 | 门派论剑、秋猎活动 |
| 冬 | 大雪封山部分区域不可去 | 内功修炼最佳时机、年末事件 |

**特殊天气（每个季节随机触发）：**

| 季节 | 天气 | 影响 |
|------|------|------|
| 春 | 细雨 | 药铺生意好（风湿药需求） |
| 春 | 春雷 | 触发特殊剧情事件 |
| 春 | 雾气 | 探索视野降低 |
| 夏 | 暴雨 | 集市客流少 |
| 夏 | 酷暑 | 体力消耗加快、茶馆火爆 |
| 夏 | 雷阵雨 | 短暂影响出行 |
| 秋 | 秋风 | 落叶景观、轻功修炼加成 |
| 秋 | 薄雾 | 探索视野略降 |
| 冬 | 大雪 | 道路封闭、适合室内修炼 |
| 冬 | 霜冻 | 药材保存时间延长 |
| 冬 | 寒风 | 体力消耗加快 |

## 五、经营系统

### 5.1 交易

- 集市买卖：物品有每日浮动价格
- NPC差异化收购：不同NPC收购价不同，信息差 = 赚钱机会
- 跑商：带货去别的地点出售（有遭遇劫匪的风险）

### 5.2 开铺子（简化版）

- 租/买一间铺子
- 选择经营类型（药铺/铁匠/茶馆）
- 进货 → 定价 → NPC自动来买
- 铺子有声望值，声望越高客流越大

### 5.3 轻量制造

- 采药 → 炼丹 → 卖丹药
- 挖矿 → 打铁 → 卖兵器
- 不需要复杂技能树，简单配方即可
- 制造可以是后期扩展内容，MVP阶段可简化或省略

## 六、NPC 系统

### 6.1 NPC 分类

| 类型 | 示例 | 功能 |
|------|------|------|
| 功能NPC | 铁匠、药铺老板、客栈掌柜 | 交易、服务 |
| 师傅NPC | 铁掌门掌门、衙门捕头 | 拜师、学艺、任务 |
| 任务NPC | 商人、村民、官差 | 悬赏、传闻、剧情 |
| 探索NPC | 路人、乞丐、神秘人 | 随机事件、彩蛋 |
| 敌对NPC | 山贼、盗匪 | 遭遇战 |

### 6.2 NPC 交互

- **对话**：选择式对话，不同选择影响好感和声望
- **交易**：买卖物品，价格受供需和好感影响
- **关系**：好感度系统，影响NPC态度和可触发事件
- **日程**：NPC有自己的作息（清晨练功、白天开店、傍晚喝酒、夜晚回家）

## 七、技术方案

### 7.1 技术栈（待定）

| 方案 | 优势 | 劣势 |
|------|------|------|
| Web（HTML/CSS/JS + Canvas/WebGL） | 跨平台、开发快、部署简单 | 性能受限 |
| Godot | 开源免费、2D支持好、导出多平台 | 学习成本 |
| Unity | 生态成熟、资源丰富 | 对2D偏重、较重 |
| Phaser.js（Web游戏框架） | 专注2D、社区活跃 | 需要JS开发 |

### 7.2 美术方案

- 使用 AI（Gemini/GPT）生成像素风素材
- 场景 Prompt：2.5D等距视角，像素风，中国古风建筑，江南水乡风格
- 角色 Prompt：32x32/64x64 像素，3/4俯视角，武侠风格NPC
- 统一调色板，保证风格一致性

## 十、风险与注意事项

| 风险 | 应对 |
|------|------|
| AI生成素材风格不一致 | 统一调色板、固定prompt模板、后期手动微调 |
| 内容量过大导致烂尾 | 严格控制MVP范围，先做核心循环 |
| 战斗系统过早设计 | MVP阶段只做文字结果，后续再设计正式战斗 |
| NPC交互设计量大 | 先做5-10个核心NPC，后续逐步扩展 |
| 经营系统复杂度失控 | 保持简化版，不引入供应链 |
