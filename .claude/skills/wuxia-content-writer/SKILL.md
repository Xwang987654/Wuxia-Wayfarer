# 武侠内容撰写规范

本技能用于为「武侠·自由人生」撰写游戏内容。所有输出必须严格遵循已设定的世界观，并具有真实的武侠质感，而非AI生成的套话。

## 使用方式

撰写任何游戏内容前，先阅读 `game-data/lore/` 下的四份世界观文档，再根据下方的铁律和风格指南输出。

撰写内容时按类型参考对应模板：
- NPC 人设 → 模板 A
- 对话文本 → 模板 B
- 任务剧情 → 模板 C
- 随机事件 → 模板 D

## 世界观铁律

以下设定不可违反，所有内容必须与之兼容。

### 朝代与天下

- 大燕朝，立国二百余年，以武定天下后偃武修文
- 如今是承平年间：表面四海升平，暗处各怀心思
- 赋税年年涨，地方官跋扈，民间怨气积攒
- 朝中党争不断，文武相轻，朝廷政令不出京城

### 地理

- 中原：天下正中，武林正宗所在，大门派扎根于此
- 江南：水乡富庶，商贸繁荣，武学传承不如中原，人重利轻义
- 塞外：苦寒之地，游牧部落，人性情彪悍，武学走刚猛路线
- 南疆：瘴气丛林，部族众多，武学多奇招怪式，中原视为旁门
- 关中：古都长安，朝廷中枢、禁军、武学总署在此
- **清风镇**：江南与中原交界，南北要道，商贸枢纽，约三百户千余人

### 势力格局

- **大门派**（2-3个）：天元宗、沧澜剑派、镇远镖局，皆在远方，根基深厚，朝廷动不了
- **小门派**：百花齐放，各地都有，规模小（十几到百人），地域性强，有独门武功
- **铁掌门**：清风镇唯一的门派，约30人，刚猛路线，掌门赵铁山60岁
- **官府武学**：标准化教学，入门快上限低，与门派武学均衡
- **武学总署**：朝廷设立，名义管理天下武学，实际想控制江湖，渗透小门派
- **青竹帮**：镇外松散组织，约20人，介于山贼和帮派之间，不滥杀

### 武学体系

- 内力与招式并重，如车之两轮
- 内力：真气凝聚，需长年累月修炼
- 招式：攻防技法，蕴含前人实战经验
- 武功等级：粗通皮毛 → 初窥门径 → 小有成就 → 登堂入室 → 炉火纯青 → 出神入化 → 返璞归真
- 门派武学：师徒传承，上限高，有独门绝学
- 官府武学：标准化，上限低，千人一面
- **清风镇水平**：赵掌门登堂入室，弟子初窥门径到小有成就，捕快粗通皮毛到初窥门径

### 朝廷与江湖

- 表面井水不犯河水
- 暗地里朝廷通过武学总署渗透小门派
- 大门派看穿但不戳破
- 部分小门派已被"扶持"为傀儡

### 清风镇规矩

- 明面：衙门管治安，铁掌门管武林事，不许在镇上动刀子
- 暗面：铁掌门说了算（赵铁山的话比捕头管用），青竹帮偶尔来镇上，夜晚客栈后院有黑市

## 风格铁律

### 绝对禁止

1. **禁止用"仿佛""宛如""似乎""犹如"做比喻的开头** — 武侠世界里的人不会随时打比方
2. **禁止用"一股XX之气涌上心头"** — 这是最典型的AI味
3. **禁止用"命运的齿轮开始转动"** — 禁止一切宿命论套话
4. **禁止用"他/她的眼中闪过一丝XX"** — 过度使用眼神描写是AI的通病
5. **禁止用"嘴角微微上扬"** — 已经被用烂了
6. **禁止用四字成语堆砌** — 一两个自然，三个以上就是AI味
7. **禁止用"深邃的目光""坚毅的面庞""挺拔的身姿"** — 模板化描写
8. **禁止用"不禁""竟然""居然"做每句的转折** — 虚词堆砌
9. **禁止用"这一刻"开头** — "这一刻，他明白了..."是AI最爱的句式
10. **禁止用"仿佛整个世界都..."** — 夸张的环境渲染

### 语言风格

**对话要像真人说话**：
- 铁匠不会说"鄙人从事锻造行业多年"，他会说"打了三十年铁了"
- 捕快不会说"维护治安是吾等职责"，他会说"这镇上的事，归我管"
- 老百姓不会说"近日物价飞涨令人忧心"，他会说"盐又涨了，这日子没法过了"

**叙述要简洁有力**：
- 不要："夕阳的余晖洒在古老的青石板路上，将一切都镀上了一层温暖的金色。"
- 要："天快黑了。集市上的摊贩开始收摊。"

**情绪要克制**：
- 不要："他心中涌起一股难以言喻的悲痛，仿佛整个世界都失去了色彩。"
- 要："他没说话，转身走了。"

**武侠世界的人有自己的逻辑**：
- 江湖人讲规矩、讲面子、讲辈分
- 普通人怕事、怕官、怕花钱
- 商人算账、算人情、算风险
- 每个人都有自己的日子要过，不是为了配合主角才存在的

### 不同身份的说话方式

**铁匠/匠人**：话少，直来直去，用具体数字。"这把剑，二十两。""不行，最少十八。"

**药铺/郎中**：谨慎，习惯用专业术语但会解释。"这味药叫半夏，有毒，但炮制过后能化痰。"

**客栈掌柜**：圆滑，见人说人话。"客官看着面生，第一次来清风镇吧？楼上请。"

**门派掌门**：有威严但不装腔作势。"拳法的事，练了才知道。光嘴上说没用。"

**衙门捕快**：公事公办，偶尔透露不满。"这事我记下了。你先回去，有消息叫你。"

**江湖散人**：说话带点江湖气，但不过分。"在下路过此地，讨碗水喝。"

**乞丐/隐士**：看似随意，偶尔说深话。"老头子我啊，年轻时候也想过闯荡江湖。后来嘛，膝盖不好，就算了。"

**商人**：三句话不离本行。"这批货从临河镇运过来，光路上就走了三天。"

**村民/普通人**：朴实，关注日常。"今年雨水好，田里收成不错。"

### 场景描写原则

- 只写玩家能看到、能互动的东西
- 不写意境，写实物。不写"岁月的痕迹"，写"墙皮剥落了大半"
- 每个地点有一两个标志性的细节就够了，不要铺陈
- 天气和时间要影响描写：雨天写泥泞，夜晚写灯火，冬天写呵气

## 模板 A：NPC 人设

```json
{
  "id": "npc_id",
  "name": "姓名",
  "role": "身份（blacksmith/doctor/innkeeper/...）",
  "age": 45,
  "personality": "用3-5个词描述性格，不要用'善良''勇敢'这种空泛的词",
  "speaking_style": "这个人说话有什么特点，举一个具体的例子",
  "background": "200字以内的背景故事。要回答：他为什么在这里？他以前做什么？他有什么遗憾？",
  "secret": "一个不为人知的秘密。不需要惊天动地，小秘密更真实",
  "wish": "一个具体的愿望。不要'世界和平'这种，要'想再喝一次老家的米酒'这种",
  "daily_quote": "这个人日常最常说的一句话",
  "location": "默认所在地",
  "schedule": {
    "0": { "location": "地点", "action": "在做什么" },
    "1": { "location": "地点", "action": "在做什么" },
    "2": { "location": "地点", "action": "在做什么" },
    "3": { "location": "地点", "action": "在做什么" },
    "4": { "location": "地点", "action": "在做什么" }
  },
  "relationships": {
    "npc_id": "关系描述（一句话）"
  },
  "shopInventory": ["物品ID"],
  "buyPreference": ["物品类型"],
  "questIds": ["任务ID"]
}
```

**撰写要点**：
- background 不要写成简历，要写成故事。有细节、有情感、有遗憾
- secret 要和这个人的身份、经历相关，不要凭空捏造
- wish 要具体到可以作为任务线索
- personality 要用行为来定义，不要用形容词。"记仇"比"性格倔强"好
- speaking_style 举的例子要直接引用这个人会说的话

## 模板 B：对话文本

```json
{
  "npc_id": "npc_id",
  "dialogues": [
    {
      "id": "dialogue_id",
      "conditions": {},
      "nodes": [
        {
          "id": "node_01",
          "text": "NPC说的话",
          "speaker": "npc",
          "choices": [
            {
              "text": "玩家的选项1",
              "effects": {},
              "next": "node_02"
            },
            {
              "text": "玩家的选项2",
              "effects": {},
              "next": "node_03"
            }
          ]
        }
      ]
    }
  ]
}
```

**撰写要点**：
- NPC 的每句台词都要符合这个人的身份和说话方式
- 对话选项不要用"正义/中立/邪恶"这种明显的设计，要用自然的语言
- 好感变化要通过对话内容自然体现，不要让玩家觉得在"刷好感"
- 一轮对话不要超过 5 个来回，NPC 不是话痨
- 对话中可以自然地传递信息（传闻、任务线索、世界观）
- NPC 有自己的事要忙，不会一直跟玩家聊下去

**对话示例**（反面教材 vs 正面教材）：

反面：
```
NPC："欢迎来到我的铁匠铺！我这里有着全镇最好的武器！请问您需要什么？"
选项：① "我想看看武器" ② "你是谁？" ③ "再见"
```

正面：
```
NPC：（头也不抬，继续打铁）"要什么自己看，架子上的都能卖。"
选项：① "这把剑怎么卖？" ② "你这有没有好一点的？" ③ （转身离开）
```

## 模板 C：任务剧情

```json
{
  "id": "quest_id",
  "name": "任务名称",
  "type": "bounty/faction/government/npc",
  "giver": "发布者NPC ID 或 notice_board",
  "description": "悬赏板上的描述（简短，一两句话）",
  "full_story": "完整的故事背景（玩家不一定能知道全部）",
  "objectives": [
    {
      "id": "obj_01",
      "type": "kill/collect/talk/go_to",
      "target": "目标ID",
      "count": 1,
      "description": "任务面板显示的描述"
    }
  ],
  "rewards": {
    "copper": 0,
    "reputation": {}
  },
  "prerequisites": {},
  "timeLimit": null,
  "flavor_text": "任务完成后的总结语（一两句话）"
}
```

**撰写要点**：
- description 是悬赏板上写的，要像真的告示：简短、直接、有悬赏金额
- full_story 是真实背景，可能和表面描述不一样（有反转）
- 任务不要都是"去杀X个怪"，要有选择空间：可以谈判、可以智取、可以无视
- rewards 要合理：清剿山贼给50两算大方了，一个小任务给几百两就离谱
- flavor_text 是玩家完成后看到的，要有收束感，不要草草了事
- 任务中涉及的 NPC 要和已有的人设一致

**任务描述示例**：

反面：
```
"勇敢的侠士，后山有一群凶恶的山贼，请前去消灭他们，保护清风镇的和平！赏银五十两。"
```

正面：
```
"后山近日不太平，有商旅被劫。能料理的来衙门找陈捕头。赏银五十两。"
```

## 模板 D：随机事件

```json
{
  "id": "event_id",
  "name": "事件名称",
  "type": "random/exploration/conditional",
  "trigger": {
    "location": "触发地点",
    "probability": 0.2,
    "season": null,
    "weather": null,
    "timeOfDay": null
  },
  "description": "事件描述（展示给玩家看的）",
  "choices": [
    {
      "text": "选项文本",
      "conditions": {},
      "effects": {},
      "result_text": "选择后的结果描述"
    }
  ],
  "cooldown": 3
}
```

**撰写要点**：
- 事件是玩家在探索中"碰上"的，要有一种偶然感
- description 要有画面感，但不要铺陈太长，三五句话够了
- 选项要有实际意义：不同选择有不同后果，不是摆设
- 不是每个事件都要打打杀杀，可以是偶遇、发现、闲聊
- result_text 要交代结果，给玩家一个交代，不要悬而未决
- 事件中出现的 NPC 如果是已有的，要符合其人设
- 冷却时间要合理：同一个人不会天天在路上等着你

**事件描述示例**：

反面：
```
"你走在竹林小径上，突然听到一阵凄厉的呼救声。你循声望去，发现一个美丽的女子正被两个凶神恶煞的山贼围住。她的眼中满是恐惧和绝望..."
```

正面：
```
"竹林里有人。一个年轻女人蹲在路边，怀里抱着个孩子，旁边站着两个汉子，看着不像善茬。"
```

## 检查清单

撰写完任何内容后，用以下清单自检：

- [ ] 是否与 `game-data/lore/` 中的世界观一致？
- [ ] NPC 的说话方式是否符合其身份？
- [ ] 是否有 AI 味的套话？（对照禁止清单逐条检查）
- [ ] 对话是否像真人说话？
- [ ] 场景描写是否简洁、只写实物？
- [ ] 任务/事件是否有选择空间？
- [ ] 数值是否合理？（金钱、声望、时间）
- [ ] 是否与已有的其他 NPC/任务/事件有矛盾？
